# Chapter 3. Introduction

This report details the work carried out for a final year project. This project was undertaken throughout the third year of the BSc Honours Combined Studies Course at The Hatfield Polytechnic.

The development of meta-assemblers has been in progress since the mid-1950's. Ferguson, in his report on the evolution of the meta-assembly program26, defines the meta-assembler.

```
           'A meta-assembly program is a machine independent assembly program. It is machine
           independent in the sense that both the assembly language which it is to accept (the source
           language) and the machine code which it is to generate (the object language) are supplied
           as part of the program to be assembled. That is, both the source language and the object
           language are input parameters to the meta-assembler.'
```

This was the first article which clearly defines the concepts of meta-assemblers. Since then, many projects have been carried out in this field. They have presented various solutions to the problem, many of which have compromised on the generality of the meta-assembler. Other terms have been introduced which describe these assemblers. They include Universal Cross-Assembler, Table-Driven Assembler and Program Meta-Assembler. For sake of simplicity, if not for absolute correctness, I will encompass all these names under the heading of meta- assembler and meta-assembly pointing out, where relevant, which class of meta-assembler is being referred to.

There were two primary objectives in conducting this project. These were:

1. To document the development of meta-assemblers from their earliest forms to present day implementations. This includes details about construction techniques, classifications, implementation methods and principles of operation.

2. To attempt a sample implementation of a meta-assembler, to gain a grasp of the problems and possible solutions to the meta-assembly concept.

The project report structure therefore reflects these two objectives. Section 4 is concerned with meta-assemblers and Section 5 is concerned with the sample implementation. Section 6 concludes the work carried out during the project and speculates on the future for meta-assemblers.
